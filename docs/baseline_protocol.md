# Baseline Protocol for the KAN-XAI Formula-Fidelity Paper

Last updated: 2026-05-25

## Main-Text Baselines

Use the following baselines in the main paper.

1. Raw tuned KAN
   - Full input space.
   - Main diagnostic for whether formula fidelity emerges without support help.

2. RF-screened tuned KAN
   - Random forest selects a low-dimensional support before KAN training.
   - Treat as a diagnostic support intervention, not as a proposed algorithm.

3. Oracle-support tuned KAN
   - True active variables are forced into the selected support.
   - Separates low-dimensional representability from full-space discovery.

4. Random-support KAN
   - Negative control for "dimension reduction alone".

5. Exclude-interaction KAN
   - Negative control that removes true interaction endpoints.
   - Tests endpoint necessity.

6. Early-stopped MLP
   - Use raw, RF-screened, oracle-support, random, and exclude-interaction versions from:
     `results/mlp_tuning/mlp_screened_summary.csv`
   - Keep this as the main MLP baseline because it is stable and does not use aggressive final refits.

## Appendix-Only Baselines

Keep these out of the main narrative unless a reviewer asks.

1. Aggressive MLP full-refit / LBFGS variants
   - Files such as `mlp_screen_specific_full_refit_summary.csv`.
   - These are useful sensitivity checks but create unnecessary tuning complexity.

2. RF predictor diagnostics
   - `results/rf_diagnostic/rf_diagnostic_summary.csv`.
   - Useful to show that RF support retention can be good while RF interaction scoring is not a clean formula-fidelity solution.

3. Feynman-style formulas
   - `results/feynman_pilot/feynman_benchmark_summary.csv`.
   - Use as external validation / appendix, not as core mechanism evidence.

4. Path deletion / path reliance
   - Useful qualitative diagnostics, but less stable than finite-difference and Hessian interaction scoring.

## Current Baseline Table

The current compact baseline table is generated by:

```bash
python experiments/make_paper_baseline_table.py
```

Output:

```text
results/paper_figures/final_baseline_table.csv
```

For the core `d=100, n=1024, c=1` setting, the key pattern is:

- Raw KAN: low MSE and strong but imperfect formula recovery.
- RF/oracle KAN: near-perfect prediction and interaction recovery.
- Random/exclude KAN: fail, confirming that support content matters.
- Raw MLP: substantially worse prediction and weaker structure recovery.
- RF/oracle MLP: can recover the interaction ranking but fits the low-dimensional formula much less accurately than screened KAN.

## Reporting Rule

In the paper, describe RF screening as a diagnostic probe:

> RF screening helps only when it preserves formula-relevant endpoints; it is not a formula-faithful solution by itself.

Avoid claiming that RF-screened KAN is a new method.
